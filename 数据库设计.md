# AI 数字人视频定制平台 - 数据库设计文档

## 一、数据库设计原则

1. **命名规范**
   - 表名：小写，下划线分隔，复数形式（如 `users`, `digital_humans`）
   - 字段名：小写，下划线分隔
   - 主键：统一使用 `id`（BIGINT, AUTO_INCREMENT）
   - 外键：`关联表名_id`（如 `user_id`, `project_id`）
   - 时间字段：`created_at`, `updated_at`, `deleted_at`

2. **字段类型规范**
   - 主键：`BIGINT UNSIGNED`
   - 外键：`BIGINT UNSIGNED` 或 `INT UNSIGNED`
   - 状态/枚举：`TINYINT` 或 `VARCHAR(20)`
   - 布尔：`TINYINT(1)` (0/1)
   - 金额：`DECIMAL(10,2)` 或 `DECIMAL(12,4)`
   - 带小数：`DECIMAL` 或 `FLOAT`
   - 文本：`TEXT` / `LONGTEXT`
   - JSON：`JSON` (MySQL 5.7+ / PostgreSQL)

3. **索引设计**
   - 主键自动建立索引
   - 外键自动建立索引
   - 频繁查询字段建立单列索引
   - 复合查询条件建立复合索引
   - 状态/时间查询建立复合索引

4. **软删除**
   - 核心数据表使用 `deleted_at` 软删除
   - 日志/审计表不使用软删除

### 2.1 用户系统模块

#### 2.1.1 用户主表 (users)

```sql
CREATE TABLE users (
    id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT COMMENT '用户ID',
    phone VARCHAR(20) UNIQUE COMMENT '手机号，登录用',
    email VARCHAR(100) UNIQUE COMMENT '邮箱，登录用',
    password_hash VARCHAR(255) NOT NULL COMMENT '密码哈希值',
    nickname VARCHAR(50) NOT NULL COMMENT '昵称',
    avatar_url VARCHAR(500) COMMENT '头像URL',
    bio TEXT COMMENT '个人简介',
    real_name VARCHAR(50) COMMENT '真实姓名',
    id_card_number VARCHAR(18) COMMENT '身份证号',
    real_name_verified TINYINT(1) DEFAULT 0 COMMENT '实名认证状态: 0-未认证, 1-已认证',

    membership_type ENUM('free', 'basic', 'pro', 'enterprise') DEFAULT 'free' COMMENT '会员类型',
    membership_expire_at DATETIME COMMENT '会员到期时间',

    quota_digital_human INT DEFAULT 3 COMMENT '数字人配额',
    quota_video_monthly INT DEFAULT 10 COMMENT '每月视频生成次数配额',
    quota_video_max_duration INT DEFAULT 60 COMMENT '单视频最长时长(秒)',
    quota_storage_mb INT DEFAULT 1024 COMMENT '存储空间配额(MB)',

    status ENUM('active', 'suspended', 'deleted') DEFAULT 'active' COMMENT '账号状态',

    last_login_at DATETIME COMMENT '最后登录时间',
    last_login_ip VARCHAR(45) COMMENT '最后登录IP',
    register_ip VARCHAR(45) COMMENT '注册IP',
    register_source ENUM('web', 'mobile', 'wechat', 'google', 'apple') DEFAULT 'web' COMMENT '注册来源',

    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted_at DATETIME COMMENT '软删除时间',

    INDEX idx_phone (phone),
    INDEX idx_email (email),
    INDEX idx_membership (membership_type, membership_expire_at),
    INDEX idx_status (status),
    INDEX idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户主表';
```

## 四、核心业务流程表数据流转

### 4.1 数字人创建流程

```
1. 用户上传照片 - 存储到 user_assets (asset_type=image)
   └─► digital_humans (status=pending, source_photos=[...])

2. 触发审核 - content_reviews (target_type=digital_human, review_type=photo_upload)
   └─► 若通过 → digital_humans (authorization_status=approved/auto_approved)
   └─► 若不通过 → digital_humans (authorization_status=rejected)

3. 创建生成任务 - digital_human_tasks (status=queued)
   └─► 入队处理 (priority: VIP=8, 普通=5)

4. 任务执行 - digital_human_tasks (status=processing, progress=0-100)
   └─► 调用 AI 模型

5. 任务完成 - digital_human_tasks (status=completed)
   └─► 更新 digital_humans
        - status=completed
        - model_file_url=...
        - preview_image_url=...
        - preview_video_url=...

6. 用户成功创建数字人
```

### 4.2 视频生成流程

```
1. 创建项目 - video_projects (status=draft)
    -- 质量与审核   - 配置:_resolution, aspect_ratio, fps
   - 选择: background_asset_id, bgm_asset_id

2. 提交生成 - video_projects (status=queued, priority=VIP:普通)
   └─► 创建 generation_tasks (task_type=full_pipeline, status=queued)

3. 任务入队 - generation_tasks (queue_position=排队位置)

4. 预处理阶段 - generation_tasks (status=preprocessing, 步骤)
   └─► TTS 语音合成
        - 生成 tts_audio_files
        - 计算实际时长

5. 生成阶段 - generation_tasks (status=generating, 步骤)
   └─► 调用视频生成 AI 模型
   - 参数: digital_human, TTS音频, 背景, 配置
   - 模型路由: 根据 priority/负载选择模型

6. 后处理阶段 - generation_tasks (status=postprocessing)
   └─► 字幕烧录
   └─► BGM混音
   └─► 水印嵌入
   └─► 视频编码

7. 质检阶段 - generation_tasks (status=quality_check)
   └─► AI 内容审核
   └─► 创建 content_reviews (target_type=video_output)

8. 任务完成 - generation_tasks (status=completed)
   └─► 创建 video_outputs
        - video_url=...
        - thumbnail_url=...
        - duration=...
        - review_status=...
   └─► 更新 video_projects
        - generation_status=completed
        - cost_cents=实际成本
   └─► 扣除用户配额
   └─► 发送通知 - user_messages (message_type=task_complete)

9. 用户收到通知, 查看视频
```

### 4.3 支付与会员流程

```
1. 用户选择套餐 - 创建 user_memberships (status=pending)
   - 生成 order_no
   - 记录: original_price, actual_price

2. 支付成功 - user_memberships (status=paid, paid_at=now)
   └─► 更新 users
        - membership_type=套餐类型
        - membership_expire_at=结束时间
        - quota_*=新的配额

3. 创建流水 - user_wallet_transactions
   - 若钱包支付: transaction_type=consume
   - 若第三方支付: payment_method=alipay/wechat
   - 记录: balance_before, balance_after

4. 发送通知 - user_messages (message_type=payment)
```
